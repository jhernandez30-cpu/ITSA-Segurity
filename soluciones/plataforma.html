Arquitectura SaaS inicial

Una plataforma separada del sitio principal

El index conserva su rol comercial. La carpeta /plataforma/ concentra login, dashboard, reportes y analítica para evolucionar luego hacia Supabase, motor real de monitoreo y pagos.

Login y registro demo

Usuarios temporales en localStorage con roles client y admin. Preparado para Supabase Auth.

Dashboard cliente

KPIs de sitios, SSL, SEO, rendimiento, alertas, historial y recomendaciones.

Reportes

Reportes demo descargables en HTML o JSON, listos para evolucionar a PDF automatico.

Excel dashboard

Tabla de revisiones y plantilla CSV compatible con Excel para reportes iniciales.

Power BI preparado

Contenedor responsive para embed seguro sin inventar enlaces reales.

Panel admin

Resumen de usuarios demo, sitios, revisiones, alertas criticas y planes activos.

Qué monitorea de forma autorizada

  • Disponibilidad, codigo HTTP y tiempo de respuesta.
  • SSL valido y senales basicas de seguridad.
  • SEO tecnico, rendimiento y estado general.
  • Hallazgos, alertas y recomendaciones para el cliente.

Limites de seguridad

La plataforma no ejecuta escaneo ofensivo, explotacion, espionaje ni vigilancia no autorizada. Las URL deben ser publicas, iniciar con http:// o https:// y pertenecer al cliente o estar formalmente autorizadas.
